Unit 5

 

1. Malicious Software: Viruses and Related Threats

  • Viruses: Programs designed to replicate and spread by attaching themselves to other programs or files. They often have malicious payloads.
  • Worms: Self-replicating programs that spread across networks without requiring a host file.
  • Trojan Horses: Programs that appear harmless but contain malicious code. They trick users into installing them.

  • Viruses:

    • Propagation: Viruses often spread through infected files, email attachments, or malicious websites.
    • Payloads: They may have destructive payloads, such as data corruption, or stealthy payloads, such as information theft.
    • Polymorphic Viruses: Change their appearance to avoid detection.

    Worms:

    • Network Spread: Worms exploit network vulnerabilities to spread rapidly.
    • Self-Replication: They independently propagate without the need for a host file.

    Trojan Horses:

    • Deceptive Nature: Trojans disguise themselves as legitimate software to trick users into installing them.
    • Backdoors: They may create backdoors for remote attackers to gain unauthorized access.

 

2. Virus Countermeasures

  • Antivirus Software: Programs designed to detect, prevent, and remove malicious software.
  • Regular Updates: Keeping antivirus software and operating systems up-to-date to protect against the latest threats.
  • User Education: Training users to be cautious about email attachments, downloads, and suspicious links.

  • Antivirus Software:

    • Signature-Based Detection: Identifies known malware patterns.
    • Heuristic Analysis: Detects previously unseen threats based on behavioral patterns.
    • Real-Time Protection: Constantly monitors for malicious activity.

    Regular Updates:

    • Virus Definitions: Regularly updating antivirus databases to recognize new threats.
    • Software Patches: Keeping operating systems and software up-to-date to patch security vulnerabilities.

    User Education:

    • Awareness Programs: Educating users about safe online practices.
    • Email Hygiene: Teaching caution regarding email attachments and links.

 

3. Distributed Denial of Service (DDoS) Attacks

  • DDoS Attacks: Overwhelming a target system or network with a flood of traffic, rendering it unavailable to users.
  • Botnets: Networks of compromised computers (zombies) used to launch coordinated DDoS attacks.
  • Mitigation: Employing techniques such as traffic filtering, rate limiting, and the use of content delivery networks (CDNs) to protect against DDoS attacks.

  • DDoS Attacks:

    • Volume-Based Attacks: Flooding the target with a high volume of traffic.
    • Protocol-Based Attacks: Exploiting vulnerabilities in network protocols.
    • Application Layer Attacks: Overwhelming web applications with requests.

    Botnets:

    • Command and Control Servers (C&C): Coordinating the actions of compromised systems.
    • Zombies: Infected computers acting as part of the botnet.

    Mitigation:

    • Traffic Filtering: Identifying and blocking malicious traffic.
    • Rate Limiting: Restricting the number of requests from a single source.
    • Content Delivery Networks (CDNs): Distributing content across multiple servers to absorb traffic.

 

4. Firewalls: Firewall Design Principles

  • Firewall: A network security device that monitors and controls incoming and outgoing network traffic based on predetermined security rules.
  • Design Principles: Determining which traffic is allowed or blocked based on security policies.
  • Stateful Inspection: Examining the context of communication to make decisions based on the state of the connection.

  • Design Principles:

    • Default-Deny Policy: Blocking all traffic by default and allowing only what is explicitly permitted.
    • Stateful Inspection: Examining the context of each connection to make informed decisions.
    • Application Layer Filtering: Filtering traffic based on application protocols.

 

5. Trusted Systems

  • Trusted Systems: Systems that meet specific security criteria and can be relied upon to enforce security policies.
  • Trusted Computing Base (TCB): The set of hardware, software, and firmware that is critical to maintaining security.
  • Evaluation Assurance Level (EAL): A numeric scale used to indicate the level of trustworthiness of security functions.

  • Trusted Systems:

    • Security Labels: Assigning labels to data and systems based on their sensitivity.
    • Multilevel Security: Managing information at multiple security levels on a single system.

    Trusted Computing Base (TCB):

    • Security Kernel: The core of the TCB responsible for enforcing security policies.
    • Reference Monitor: A security mechanism ensuring that every access is authorized.

    Evaluation Assurance Level (EAL):

    • Scale Levels: Ranging from EAL1 (functionally tested) to EAL7 (formally verified design).

 

6. Common Criteria for Information Technology Security Evaluation

  • Common Criteria (CC): An international standard (ISO/IEC 15408) for evaluating and certifying the security features of IT products.
  • Security Target (ST): A document that defines the security requirements for a specific IT product.
  • Protection Profiles (PP): Documents that specify security requirements for a particular type of IT product.

  • Common Criteria (CC):

    • International Standard: ISO/IEC 15408 for evaluating and certifying the security features of IT products.
    • Collaborative Framework: Used by multiple countries to establish a common baseline for security evaluation.

    Security Target (ST):

    • Document Definition: Specifies the security requirements for a specific IT product.
    • Scope and Environment: Describes the intended use and security environment.

    Protection Profiles (PP):

    • Document Specification: Specifies security requirements for a particular type of IT product.
    • Tailoring: Allows adjustments based on the specific needs of the product.
