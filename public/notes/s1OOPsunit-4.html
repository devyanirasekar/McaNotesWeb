Unit-4 :

Polymorphism


- A C++ virtual function is a member function in the base class that you redefine
in a derived class. It is declared using the virtual keyword.
- A &#39;virtual&#39; is a keyword preceding the normal declaration of a function.
- C++ determines which function is to be invoked at the runtime based on the
type of the object pointed by the base class pointer.
- It is used to tell the compiler to perform dynamic linkage or late binding on the
function.
- Functions are declared with a virtual keyword in base class
When you refer to a derived class object using a pointer or a reference
to the base class, you can call a virtual function for that object and
execute the derived class’s version of the function. 
- Virtual functions ensure that the correct function is called for an object,
regardless of the type of reference (or pointer) used for function call.
- They are mainly used to achieve Runtime polymorphism
- The resolving of function call is done at runtime.
- Runtime polymorphism is achieved only through a pointer (or
reference) of base class type.
- A base class pointer can point to the objects of base class as well
as to the objects of derived class.
-  base class pointer ‘base_class_objecr’ contains the address of
object ‘derived_class_object ’ of derived class.
- Using pointer to create base class pointer and using &amp; operator
to hold the address of derived class objects and using late
binding of base class objects with method name.
int main()    
{    
 A* a;    //pointer of base class    
 B b;     //object of derived class    
 a = &amp;b;    //&amp; optr using to hold the method of address
 a-&gt;display();   //Late Binding occurs    
}  
Rules for Virtual Functions
 The functions cannot be static
 You derive them using the “virtual” keyword
 Virtual functions in C++ needs to be a member of some other class (base
class)
 They can be a friend function of another class
 The prototype of these functions should be the same for both the base and
derived class
 Virtual functions are accessible using object pointers
 Redefining the virtual function in the derived class is optional, but it needs
to be defined in the base class
 The function call resolving is done at run-time
 You can create a virtual destructor but not a constructor
Syntax:
Class Base {
public:
Virtual void print() {
// code
}
};
class Derived : public Base {
public:
void print() {
// code
}
};
 

Pure Virtual Functions and Abstract Classes in C++

Pure Virtual Function


o A pure virtual function is a virtual function in C++ for which we need not to write
any function definition and only we have to declare it.
o It is declared by assigning 0 in the declaration.
o A virtual function is not used for performing any task. It only serves as a
placeholder.
o When the function has no definition, such function is known as &quot;do-nothing&quot;
function.
o The &quot;do-nothing&quot; function is known as a pure virtual function. A pure virtual
function is a function declared in the base class that has no definition relative
to the base class.
o A class containing the pure virtual function cannot be used to declare the
objects of its own, such classes are known as abstract base classes.
The main objective of the base class is to provide the traits to the derived
classes and to create the base pointer used for achieving the runtime
polymorphism.
o A virtual function in C++ is a base class member function that you can redefine in a
derived class to achieve polymorphism.
o You can declare the function in the base class using the virtual keyword. 
o Once you declare the function in the base class, you can use a pointer or reference to
call the virtual class and execute its virtual version in the derived class.
Pure virtual function can be defined as:
1. virtual void display() = 0;   
Let&#39;s see a simple example:
#include &lt;iostream&gt;
using namespace std;
class Base{
public:
virtual void Output(){
cout &lt;&lt; &quot;Output Base class&quot; &lt;&lt; endl;
}
void Display(){
cout &lt;&lt; &quot;Display Base class&quot; &lt;&lt; endl;
}
};
class Derived : public Base{
public:
void Output(){
cout &lt;&lt; &quot;Output Derived class&quot; &lt;&lt; endl;
}
void Display()
{
cout &lt;&lt; &quot;Display Derived class&quot; &lt;&lt; endl;
}
};
int main(){
Base* bpointr;
Derived dpointr;
bpointr = &amp;dpointr;
// virtual function binding
bpointr-&gt;Output();
// Non-virtual function binding
bpointr-&gt;Display();
}
 

Abstract base classes


- A class that contains a pure virtual function is known as an abstract
class. In the above example, the class Shape is an abstract class.
- An abstract class is a class that is designed to be
specifically used as a base class.
- An abstract class contains at least one pure virtual
function. You declare a pure virtual function by using a
pure specifier ( = 0 ) in the declaration of a virtual
member function in the class declaration.
- Abstract classes are used to express broad concepts from which more
concrete classes can be derived. 
-  An abstract class type object cannot be created.
- To abstract class types, however, you can use pointers and references.
- Declare at least one pure virtual member feature when creating an abstract class.
The pure specifier (= 0) syntax is used to declare a virtual function.
